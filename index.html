Sample Chapter

Chapter 5: Control Flow in Rust

Download Full Sample 
// Example: Using match with enums
enum Direction {
    North,
    South,
    East,
    West,
}

fn describe_direction(dir: Direction) {
    match dir {
        Direction::North => println!("Heading north!"),
        Direction::South => println!("Going south!"),
        Direction::East => println!("Eastward bound!"),
        Direction::West => println!("Westward ho!"),
    }
}

Rust's control flow mechanisms provide powerful ways to direct your program's execution path. In this chapter, we'll explore conditional statements, loops, pattern matching, and early returns.

Unlike many other languages, Rust treats if expressions as true expressions that return values. This approach allows for concise, functional-style code that's both readable and expressive.

Key Insight

Rust's match expression is exhaustive, meaning it requires you to handle all possible cases. This feature prevents many common bugs found in other languages and is one of Rust's safety guarantees.
